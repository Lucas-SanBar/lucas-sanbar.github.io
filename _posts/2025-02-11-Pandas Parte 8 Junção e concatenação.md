No pandas, os m√©todos `merge()` e `concat()` s√£o opera√ß√µes fundamentais para se trabalhar com mais de um DataFrame. Assim como no SQL as opera√ß√µes de `join` e `union` est√£o entre as mais utilizadas.

## Merge

No pandas, o `merge()` √© usado para combinar dois DataFrames, funcionando de maneira semelhante a um `JOIN` em **SQL**. Ele √© feito com o m√©todo `merge()`, que permite unir os dados com base em uma ou mais colunas.

Sintaxe b√°sica do m√©todo `merge()` :

`df_merge = pd.merge(DataFrame1, DataFrame2, on='coluna_chave', how='tipo_de_join')`

* **DataFrame1**, **DataFrame2** : DataFrames que ser√£o √∫nidos;
* **on** : Coluna chave utilizada na jun√ß√£o;
* **how**: Tipo de jun√ß√£o;

Tipos de Merge

| Tipo de `merge()` | Descri√ß√£o                                                                                               |
| ----------------- | ------------------------------------------------------------------------------------------------------- |
| `inner` (padr√£o)  | Mant√©m apenas os registros correspondentes nas duas tabelas.                                            |
| `left`            | Mant√©m todos os registros do primeiro DataFrame e os correspondentes do segundo.                        |
| `right`           | Mant√©m todos os registros do segundo DataFrame e os correspondentes do primeiro.                        |
| `outer`           | Retorna todos os registros de ambos os DataFrames, preenchendo com NaN onde n√£o houver correspond√™ncia. |
| `cross`           | Retorna o produto cartesiano de ambos os DataFrmes.                                                     |

### Inner Join

O **INNER JOIN** retorna apenas as linhas que t√™m correspond√™ncia entre os DataFrames. Se uma chave estiver presente em apenas um dos DataFrames, a linha correspondente ser√° descartada.

![inner join](/assets/images/11-02-2025/Inner Join.png)
_Exemplo do `inner join`_

Utilizaremos os DataFrames criados `df_alunos` e `df_notas`.

```python
df_inner_merged = pd.merge(df_alunos, df_notas, on='id_aluno')

```

![Bloco 3](/assets/images/11-02-2025/Bloco 3.png)

Como o **inner join** j√° √© a jun√ß√£o padr√£o do Pandas, n√£o √© necess√°rio especificar no atributo `how=`

Perceba que:
* O **inner join** mant√©m apenas as correspond√™ncias entre os DataFrames;
* As linhas sem correspond√™ncia s√£o exclu√≠das;
* Podemos unir com uma ou mais colunas-chave (Podemos passar no atributo `on=` uma lista de colunas chaves);

Portanto, nosso DataFrame com o `merge(merge(df_alunos, df_notas, on='id_aluno')` feito possui no final apenas **12 linhas**. Pois, o limitante da jun√ß√£o √© o DataFrame com menor √≠ndice `df_notas`.

Quando usar o **inner join**:
* Quando queremos apenas os dados que possuem correspond√™ncia entre os dois DataFrames;
* Para limpar dados, removendo registros sem correspond√™ncia;

### Left Join

O **LEFT JOIN** retorna todas as linhas do primeiro DataFrame (DataFrame 1 na imagem), mantendo apenas as correspond√™ncias do segundo DataFrame (DataFrame 2 na imagem). Se n√£o houver correspond√™ncia, os valores do segundo DataFrame (DataFrame 2) ser√£o preenchidos com `NaN`.

![left join](/assets/images/11-02-2025/Left Join.png)
_Exemplo do `Left join`_

Utilizaremos os DataFrames criados `df_alunos` e `df_notas`.

```python
df_left_merged = pd.merge(df_alunos, df_notas, on='id_aluno', how='left')

df_left_merged
```

![Bloco 4](/assets/images/11-02-2025/Bloco 4.png)
_DataFrame criado `df_left_merged` contendo o left join_

Perceba que:
* O **left join** mant√©m as linhas do primeiro DataFrame (`df_alunos`) intacta;
* As linhas sem correspond√™ncia no segundo DataFrame (`df_notas`) s√£o preenchidos com `NaN`;
* Podemos unir com uma ou mais colunas-chave (Podemos passar no atributo `on=` uma lista de colunas chaves);

Portanto, nosso DataFrame com o `merge(df_alunos, df_notas, on='id_aluno', how='left')` feito possui no final todas as **20 linhas** do DataFrame `df_alunos`. Pois, o limitante da jun√ß√£o √© sempre o primeiro DataFrame.

Quando usar o **left join**:
* Quando queremos manter todos os registros do primeiro DataFrame, mesmo sem correspond√™ncia no segundo;
* Quando estamos trabalhando com tabelas principais e secund√°rias, onde o DataFrame 1 cont√©m todos os registros e o DataFrame 2 tem apenas informa√ß√µes adicionais;
* Quando queremos evitar perda de dados ao unir tabelas;

* As linhas sem correspond√™ncia no primeiro DataFrame (`df_alunos`) s√£o preenchidos com `NaN`;
* Podemos unir com uma ou mais colunas-chave (Podemos passar no atributo `on=` uma lista de colunas chaves);

Portanto, nosso DataFrame com o `merge(df_alunos, df_notas, on='id_aluno', how='right')` feito possui no final todas as **16 linhas** do DataFrame `df_notas`. Pois, o limitante da jun√ß√£o √© sempre o segundo DataFrame.

Quando usar o **right join**:
* Quando queremos manter todos os registros do segundo DataFrame, mesmo sem correspond√™ncia no primeiro;
* Quando estamos trabalhando com tabelas principais e secund√°rias, onde o DataFrame 2 cont√©m todos os registros e o DataFrame 1 tem apenas informa√ß√µes adicionais;
* Quando queremos evitar perda de dados do segundo DataFrame ao unir tabelas;

O **OUTER JOIN** retorna todas as linhas de ambos os DataFrames, mantendo as correspond√™ncias sempre que poss√≠vel. Se um valor n√£o tiver correspond√™ncia no outro DataFrame, ele ser√° preenchido com `NaN`.

Perceba que:
* O **outer join** mant√©m todas as linhas de ambos os DataFrames;
* Se n√£o houver correspond√™ncia em um dos DataFrames, os valores s√£o preenchidos com `NaN`;
* Podemos unir com uma ou mais colunas-chave (Podemos passar no atributo `on=` uma lista de colunas chaves);

Portanto, nosso DataFrame com o `merge(df_alunos, df_notas, on='id_aluno', how='outer')` feito possui no final todas as **20 linhas** do DataFrame `df_notas` e todas as **16 linhas** do DataFrame `df_notas` com suas respectivas combina√ß√µes e o valor `NaN` quando n√£o existe correspond√™ncia.

Quando usar o **outer join**:
* Quando queremos manter todos os registros de ambos os DataFrames, independentemente de haver correspond√™ncia;
* Quando estamos trabalhando com dados de fontes diferentes, e queremos unir tudo sem perder informa√ß√µes;
* Quando precisamos identificar quais registros n√£o t√™m correspond√™ncia entre os dois DataFrames;

* N√£o cont√©m o par√¢metro `on=`

Quando usar o **cross join**:
* Quando precisamos combinar todas as possibilidades entre dois conjuntos de dados;
* Em casos de simula√ß√µes e modelagens, onde cada vari√°vel precisa ser combinada com todas as outras;
* Para criar combina√ß√µes exaustivas entre duas tabelas que n√£o possuem uma chave comum;

## Concat

No **SQL**, a opera√ß√£o `UNION` combina os resultados de duas tabelas. No Pandas, usamos a fun√ß√£o `concat()` para empilhar DataFrames de forma **VERTICAL** (Adicionando mais linhas) ou de forma **HORIZONTAL** (Adicionando mais colunas).

Sintaxe b√°sica do m√©todo `concat()` :

`df_concat = pd.concat(lista_df, axis=0, join='tipo_de_join', ignore_index=False, keys=None)`

* **lista_df** : Lista de DataFrames a serem concatenados;
* **axis=0 (Padr√£o)** : Concatena verticalmente (adiciona linhas, igual ao `UNION ALL` do **SQL**);
* **axis=1** : Concatena horizontalmente (adiciona colunas, similar a um `JOIN`);
* **join='outer' (Padr√£o)** : Mant√©m todas as colunas, preenchendo `NaN` quando necess√°rio;
* **join='inner'** : Mant√©m somente as colunas comuns entre os DataFrames;
* **ignore_index=True** : √çndices v√£o ser reajustados automaticamente;
* **keys=['R1','R2']** : Adiciona r√≥tulos de identifica√ß√£o para cada DataFrame concatenado;

Quando usar a **concatena√ß√£o com `axis=0`?**:
* Quando temos dados adicionais da mesma estrutura e queremos empilha-los;
* Para combinar datasets de diferentes per√≠odos (ex: dados de vendas mensais e/ou anuais);
* Para empilhar registros de diferentes fontes sem alterar as colunas;

* Quando temos dados complementares sobre as mesmas observa√ß√µes;
* Quando queremos juntar informa√ß√µes de diferentes fontes;
* Quando precisamos combinar colunas extras em um DataFrame;
